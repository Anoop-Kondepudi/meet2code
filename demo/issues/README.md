# Known Issues

This folder documents intentional bugs and missing features in the TodoApp demo. These issues exist so that the Meet2Code agentic pipeline can detect and fix them during demo meetings.

| # | Title | Label | Severity | Component | Description |
|---|-------|-------|----------|-----------|-------------|
| 001 | Dark Mode Toggle | feature | Medium | frontend | No dark mode support or toggle button |
| 002 | Edit Todo | feature | Medium | frontend | Cannot edit/rename todos after creation |
| 003 | Due Dates | feature | Medium | both | No due date functionality |
| 004 | Priority Levels | feature | Medium | both | No priority indicators |
| 005 | Search & Filter | feature | Medium | frontend | No search or filter capabilities |
| 006 | Persistent Storage | bug | High | backend | Data lost on server restart |
| 007 | Delete Confirmation | improvement | Low | frontend | No confirmation before deleting |
| 008 | Empty State | improvement | Low | frontend | Blank space when no todos exist |
| 009 | Long Text Overflow | bug | Medium | frontend | Long titles break card layout |
| 010 | Hover States | improvement | Low | frontend | Buttons lack hover/active feedback |
| 011 | Keyboard Shortcuts | improvement | Low | frontend | Enter key doesn't submit new todo |
| 012 | Accessibility Labels | improvement | Low | frontend | Interactive elements lack aria-labels |
